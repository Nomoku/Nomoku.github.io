初学python

2018-10-02

前言

近几年，python这门语言一直牢牢占据着TIOBE排行榜的前五名。但一直没有去做过多的了解。这学期学校开的一门名为《利用Python进行数据分析》的课程。我才真正的开始接触python。学习的时日不长但是python给我的感觉和其他语言却完全不同（一个百依百顺的小老婆（误））。

python的前世今生

Python的创始人为Guido van Rossum。之所以选中Python（大蟒蛇的意思）作为该编程语言的名字，是因为他是一个叫Monty Python的喜剧团体的爱好者。
1991年，第一个Python编译器诞生。它是用C语言实现的，并能够调用C语言的库文件。从一出生，Python已经具有了：类，函数，异常处理，包含表和词典在内的核心数据类型，以及模块为基础的拓展系统。
而如今python已经发行到3.5版本，用丰富的库对其功能进行扩展，并在全球范围内拥有许多忠实的拥簇。

应用领域

课本上以及我在网络上的了解，python涉及的领域非常多，基本上主流的计算机研究方向都有涉猎。

  1. 大数据：python在爬虫程序上有着不可撼动的霸主地位，同时Numpy/matplotlib/Scipy等库的扩展，使得python在数据挖掘和分析数据上有着得天独厚的优势。
  2. web开发：Python拥有很多免费数据函数库、免费web网页模板系统、以及与web服务器进行交互的库，可以实现web开发，搭建web框架，目前比较有名气的Python web框架为Django。可以很简单的进行web的开学习。
  3. 桌面软件开发：Python在图形界面开发上很强大，可以用tkinter/PyQT框架开发各种桌面软件。
  4. 自动化运维：Python已经成为，运维人员必备语言，尤其是在Linux运维方面，基本上是自动化运维。

  5. etc..

    python的优缺点

    我的知识水平和编程经验不太够，所以没有什么很本质的优劣评比，以下只是我使用这语言的个人体会。

    优点

  6. 简单易学：python的基础语法非常简单，只需要对语言稍微有些了解即可，非常适合编程初学者。比如创建变量时，可以不作类型声明，而是根据赋值值的类型自动标记。甚至还有官方的中文python文档。而且作为一门高级语言，python的代码非常简洁，可阅读性非常高，就像读一篇英语文章。

  7. 丰富的库：Python标准库很庞大，而且python还有可定义的第三方库可以使用。我课上进行数据分析所使用的功能都是由第三方库提供的，Numpy/IPython等。只要安装了Python，所有这些功能都是可用的，这被称作Python的“功能齐全”理念。
  8. 扩展嵌入性强：python可以编写各种脚本来嵌入其他语言所编写的程序中，为源程序提供功能的扩展。也可以将其他语言代码组合成程序运行，做’润滑油与粘合剂‘。
  9. 开源免费：Python是FLOSS之一。由于它的开源本质，Python已经被移植在许多平台上。如果小心地避免使用依赖于系统的特性，那么所有Python程序无需修改就可以在任何平台上面运行。
  10. 应用领域广：可以使用 Python 做很多的事情。例如，Web 开发、自动化运维、数据分析、科学计算、人工智能、机器学习等等。Python 语言介于脚本语言和系统语言之间，我们根据需要，既可以将它当做一门脚本语言来编写脚本，也可以将它当做一个系统语言来编写服务。

缺陷

Every coin has two sides.
python亦有着许多不足

  1. 格式：由于python的语法相较其他语言，取消的不少符号分隔的部分，如python中取消了{}的代码分块，而是用4个空格或一个制表符分隔，所以有些代码无法写成一行。虽然齿轮代码是好习惯，阅读性强，但是需要在.py文件中书写，无法使用shell程序。
  2. 加密：python的开源性是的Python语言不能加密，所以使用python开发的网站，程序无法保证保密性。无法做到版权保护。
  3. 运行速度：Python 的运行速度相比C语言慢很多，跟JAVA相比也要慢一些，但其实这里所指的运行速度慢在大多数情况下我们是无法直接感知到的，必须借助测试工具才能体现出来。
  4. 版本不兼容：python2与python3，这会使的应用程序无法做到向上兼容。

    总结

    python作为现阶段的热门语言，发展前景非常好，很适合作为程序员的主修语言和辅修语言。以上是我在听课，网络，书籍以及自己使用python语言的体会。

初见Markdown

2018-09-04

前言

Markdown是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式。
为了搭建自己的博客，这几天拾起了以前学习过的Markdown语言，在我忘得差不多的情况下，这也算是初见吧（笑）。

语法总结

一. 标题与分隔线

Markdown 支持两种格式的标题，类Setext和类atx形式。

  1. 类Setext

    使用底线的形式，在标题下方使用=符号（最高阶标题），使用-符号（第二级标题）。

    例如

    标题H1
    =========

    效果为

    标题H1

    任何数量的=和-号都可以。

  1. 类atx

    在标题前插入1~6个#，产生1~6阶标题。

    例如

    # H1
    ## H2
    ###### H6

    效果为

    H1

    H2

    H5

在Markdown中在一行中用三个以上的星号*、减号-、底线来建立一个分隔线，行内不能有其他东西。

值得注意的是，普通文本的下一行不能接—号，否则会认为该文本为H2标题（类Setext），需要一行空行分隔

二. 段落与换行

  1. 换行
    在段末敲入两个空格换行。（转换为HTML的<\br>标签）

  2. 空行
    在Markdown源文件中无论空多少行，在渲染之后都只显示一行空行

  3. 段落
    在markdown语言中,唯一决定两行文字是否是段落的,就在于这两行文字之间是否有空行.如果这两行文字之间,有空行了,就代表,这两行文字为两个段落,如果这两行文字之间,没有空行,仅仅换行,就代表这两行文字是属于同一个段落.即使是在一行文字中的末尾,添加了两个空格之后换行,这两个行文字依旧是一个段落.

三. 区块应用

Markdown标记区块引用类似于email的引言部分，在所引用的部分，每行最前方加上>，亦可以整个段落前加上>符号。

例如

Markdown的引用部分
每一行
或一段前
都加上>符号

引用可以嵌套使用

你以为我是引用

其实我是引用的引用da~

而且区块引用中也可以使用其他的Markdown语法

四. 列表

Markdown的列表分为有序列表和无序列表

  1. 无序列表
    使用*，-，+作为列表标记
    例如

    + 列表1
    + 列表2

    效果为

    • 列表1

    • 列表2

      以上三组效果相同，但是，由于*，-容易与标题，斜体加粗等语法混淆，一般使用+作为无序列表的记号

      标号后有一个空格。

  2. 有序列表
    使用数字加一个英文句号
    标号上的数字不会影响输出，但是为了逻辑严谨，最好规范数字的使用。建议最好还是从按顺序开始，因为 Markdown 未来可能会支持有序列表的 start 属性。

    标记一般放在最左边，但也可以缩进（最多3个空格）。
    标记后一定要加一个空格。

    列表项目可以包含多个段落，每个项目下的段落都必须缩进 4 个空格或是 1 个制表符。

    如果要放代码区块的话，该区块就需要缩进两次，也就是 8 个空格或是 2 个制表符

    在行首出现数字-句点-空白，要避免这样的状况，可以在句点前面加上反斜杠\，避免出现列表判定。

五. 代码

  1. 代码区段
    如果要标记一段行内代码，可以用反引号`将其包起来（tab键上方）。

    • 如果要在代码区段内插入反引号，可以用多个反引号来开启和结束代码区段。

    • 代码区段的起始和结束端都可以放入一个空白，起始端后面一个，结束端前面一个。

    • 在代码区段内， & 和方括号都会被自动地转成 HTML 实体，这使得插入 HTML 原始码变得很容易。

  2. 代码区块
    代码区块是指和程序相关的写作或是标签语言原始码通常会有已经排版好的区块。这些区块通常不会以一般的方式去排版，而是按照原来的样子显示。

    • 创建一个代码区块只需要1个制表符或4个空格(列表中是2个/8个)。

    • 代码区块会持续到取消缩进的那一行

    • 代码区块中，一般的 Markdown 语法不会被转换。

六. 链接与图片

  1. 链接
    有行内式与参考式之分（链接文字皆以方括号来标记）

    1. 行内式
      在[ ]后面的圆括号（）中插入网址，若还想加入链接的title文字，在（）中加入，并用“”包起来

      例如

      [百度]（http://www.baidu.com/ “搜索引擎”）

      效果为

      百度

      若要链接到本地，可以使用相对地址

    2. 参考式
      类似于行内式，但（）用第二个[]和当中的标号代替，标号可以有多个单词

      例如

      [百度] [id]

      在文件的任意处，你可以把这个标记的链接内容定义出

      [id]:http://www.baidu.com/ "搜索引擎"

      效果为

      [百度] [id]
      [id]:http://www.baidu.com/ “搜索引擎”

      其中，链接内容定义形式为：

      • 方括号（前面可以选择性地加上至多三个空格来缩进），里面输入链接文字
      • 接着一个冒号
      • 接着一个以上的空格或制表符
      • 接着链接的网址,可以用<>包括
      • 选择性地接着 title 内容，可以用单引号、双引号或是括弧包着，可以放到第二行
      • 标号不区分大小写

  2. 隐式链接标记
    可以让你可以省略指定链接标记，这种情形下，链接标记会视为等同于链接文字，要用隐式链接标记只要在链接文字后面加上一个空的方括号。
    例如

    [百度] []

    [百度]:http://www.baidu.com/

    效果为

    [百度] []

    [百度]:http://www.baidu.com/

    参考式比较好读，并且产生的html文本最少

  3. 自动链接
    Markdown 支持以比较简短的自动链接形式来处理网址和电子邮件信箱，只要是用方括号<>包起来， Markdown 就会自动把它转成链接。一般网址的链接文字就和链接地址一样

    可以用来保护邮箱，糊弄一些不好的邮址收集机器人

  4. 图片
    Markdown用来标记图片的语法和链接很相似，也分为行内式和参考式，在链接前加个!（必须是英文输入法的!）

    ![Nomoku](http://...)
    但是这种方式添加本地图片不方便

    如果你的markdown在一个文件目录下，需>要添加另一个目录下的图片，绝对路径是不可行的。需要 “迂回”
    所谓 迂回，即需要先用../../命令返回上一文件目录，直至可以顺利找到要添加图片的目录。

    举个栗子
    比如你的markdown在~/Document/mymarkdown/test下，需要添加~/Downloads/Pic/background目录下的sunlight.jpg
    你需要做的是

    ![](../../../Downloads/Pic/>background/sunlight.jpg)

    还有快速获取图片链接的方法

    1. 将图片置于md文件所在的目录下，直接![](pic_name.jpg)即可。（或者在md文件所在目录下建立pic文件夹，![](pic/pic_name.jpg)）
    2. 网上的图片直接在新标签打开图片，复制地址框即为图片地址。

七. 其他

  1. 强调
    使用或者_作为标记强调的符号，一个或_包围是斜体，两个是加粗，三个及以上是斜体加粗

    注意，从一而终，并且*或_两端不能有空格。

  2. 反斜杠
    转义效果
    支持一下符号：

    • \ 反斜线
    • ` 反引号
      • 星号
    • _ 底线
    • {} 花括号
    • [] 方括号
    • () 括弧

    • 井字号

      • 加号
      • 减号
    • . 英文句点
    • ！惊叹号
